## Что такое план C простыми словами

Представь, что ты собрал 5 000 оценок от студентов: «Снимок A лучше снимка B по резкости», «Снимок C равен снимку D по контрасту» и так далее. У тебя есть рейтинг каждого изображения по каждому критерию (благодаря Elo / Bradley-Terry из этапа 5).

**Вопрос:** Можно ли научить компьютер предсказывать этот рейтинг для *новых* изображений, которые люди ещё не оценивали?

**Ответ:** Да, если данных достаточно. Это и есть план C.

### Аналогия

Представь, что ты учишь ребёнка отличать хорошие фотографии от плохих. Ты показываешь ему сотни пар: «Вот эта лучше, а вот эта хуже». Через какое-то время ребёнок начинает сам понимать, какие фотографии «хорошие» — он выучил закономерности (резкие контуры = хорошо, шум = плохо, и т.д.).

Нейросеть работает так же: она «смотрит» на пиксели изображения, извлекает из них числовые признаки (features), и учится по собранным оценкам предсказывать, какое изображение человек бы выбрал как лучшее.

## Три уровня сложности ML-компонента

### Уровень 1: Классические метрики (без обучения)

Это не машинное обучение в полном смысле, но полезная отправная точка.

**Суть:** Берём готовые алгоритмы оценки качества изображений и сравниваем их результаты с человеческими оценками. Если корреляция высокая — алгоритм «согласен» с людьми.

**Алгоритмы:**

- **BRISQUE** (Blind/Referenceless Image Spatial QUality Evaluator) — анализирует статистику яркостей пикселей. «Естественные» изображения имеют определённое статистическое распределение, а искажённые — отклоняются от него. Не требует эталонного изображения.

- **NIQE** (Natural Image Quality Evaluator) — похож на BRISQUE, но не требует обучения вообще. Использует только статистическую модель «чистых» изображений.

**Что делаем:**
1. Считаем BRISQUE/NIQE-оценку для каждого изображения в датасете (Python, библиотека `pyiqa`)
2. Считаем корреляцию (Spearman, Pearson) между BRISQUE-оценкой и рейтингом, полученным от людей
3. Рисуем scatter-plot: ось X — человеческий рейтинг, ось Y — оценка алгоритма

**Трудозатраты:** 2–3 дня
**Что это даёт для диплома:** Раздел «Сравнение субъективных и объективных метрик качества» — это сильный аналитический раздел.

### Уровень 2: Обучение регрессора на признаках CNN

**Суть:** Берём предобученную нейросеть (например, EfficientNet), прогоняем через неё каждое изображение и получаем вектор признаков (feature vector) — набор чисел, описывающих содержимое изображения. Затем обучаем простую модель (линейную регрессию или маленькую нейросеть) предсказывать рейтинг по этим признакам.

**Пошагово:**

```
[Изображение 640×480] 
    → EfficientNet (предобученная, замороженная)
    → Вектор признаков [1280 чисел]
    → Маленькая нейросеть (2 слоя)
    → Предсказанный рейтинг [1 число]
```

**Что значит «предобученная»?** EfficientNet уже обучена на миллионах изображений (ImageNet) распознавать объекты. Она «понимает» текстуры, формы, контрасты. Мы не переобучаем её — просто используем как «извлекатель признаков».

**Что значит «регрессор»?** Это модель, которая по набору чисел (признаков) предсказывает одно число (рейтинг). Например: «у этого изображения много шума (признак #42 высокий) и низкий контраст (признак #107 низкий) → предсказанный рейтинг = 1230».

**Функция потерь:**
Используется **margin ranking loss** — модель учится не предсказывать абсолютный рейтинг, а правильно упорядочивать пары:

```
loss = max(0, -sign * (score_A - score_B) + margin)
```

Если человек сказал «A лучше B», а модель дала score_A < score_B — штраф. Если порядок совпал — штрафа нет.

**Трудозатраты:** 1–2 недели
**Что нужно:** Python, PyTorch, GPU (можно Google Colab бесплатно), минимум 2000–3000 собранных сравнений

### Уровень 3: RankNet — полноценная нейросеть ранжирования

**Суть:** Вместо двухэтапного подхода (признаки → регрессор) обучаем одну нейросеть end-to-end. Она принимает изображение и выдаёт «оценку качества». Обучение происходит на парах: сеть учится давать более высокую оценку тому изображению, которое люди выбрали как лучшее.

**Архитектура RankNet:**

```
Изображение A → [CNN] → score_A ─┐
                                   ├─→ P(A > B) = σ(score_A - score_B)
Изображение B → [CNN] → score_B ─┘

Та же самая CNN (общие веса) — «сиамская» архитектура
```

CNN — одна и та же сеть с общими весами. Она обрабатывает оба изображения по очереди. Разница оценок проходит через сигмоиду σ и даёт вероятность «A лучше B». Обучение минимизирует кросс-энтропию между предсказанной и реальной (от людей) вероятностью.

**Что значит «сиамская»?** Это не две разные сети, а одна, которая применяется дважды — к левому и правому изображению. Это гарантирует, что одно и то же изображение всегда получает одну и ту же оценку, независимо от того, слева оно или справа.

**Трудозатраты:** 3–4 недели
**Что нужно:** Всё то же + минимум 5000 сравнений, GPU с >6 ГБ видеопамяти

## Активное обучение (Active Learning Loop)

Это самая интересная часть плана C, и она связывает ML-модель обратно с системой сбора данных.

**Проблема:** Собрать 5000+ качественных сравнений — дорого (время студентов). Как минимизировать число сравнений, не потеряв в качестве модели?

**Решение:** Пусть модель сама выбирает, какие пары показать человеку. Модель выбирает пары, в которых она наименее уверена — где предсказанная вероятность P(A > B) ≈ 0.5 (то есть модель «не знает», кто лучше).

**Цикл:**

```
1. Модель обучена на текущих данных
2. Модель предсказывает P(A > B) для всех непоказанных пар
3. Выбираем пару с P ≈ 0.5 (максимальная неопределённость)
4. Показываем эту пару оценщику
5. Записываем ответ, дообучаем модель
6. Повторяем с шага 2
```

**Почему это эффективно:** Случайный выбор часто показывает «очевидные» пары (отличное vs. ужасное изображение), которые не несут полезной информации. Активное обучение фокусируется на «сложных» случаях, где каждое сравнение максимально информативно.

## Интеграция ML с текущим проектом

Если решишь реализовать план C, понадобится второй сервис на Python:

```
┌──────────────────────────┐     ┌─────────────────────────┐
│   PHP-бэкенд (web/)     │     │  Python ML-сервис       │
│                          │     │                         │
│  - Отдаёт изображения    │◄───►│  - Считает BRISQUE/NIQE │
│  - Принимает оценки      │     │  - Обучает RankNet      │
│  - Хранит данные (SQLite)│     │  - Выбирает пары (AL)   │
│  - Отдаёт фронтенд       │     │  - REST API (FastAPI)   │
└──────────────────────────┘     └─────────────────────────┘
         ▲                                ▲
         │          HTTP API              │
         └────────────────────────────────┘
```

PHP вызывает Python-сервис через HTTP для:
- Получения «умной» пары (вместо random / Swiss)
- Получения предсказанного рейтинга для новых изображений
- Пересчёта модели после накопления новых данных

- Уровень 1 — **реалистичен** (нужно ~500 сравнений, это 1–2 часа работы одного оценщика)
- Уровень 2 — **выполним** при 3–5 оценщиках, каждый делает ~200 сравнений за сессию
- Уровень 3 — **амбициозен**, но возможен при 5–10 активных оценщиках

## Научные источники

### Основы ранжирования на основе попарных сравнений

1. **Thurstone, L. L.** (1927). A law of comparative judgment. *Psychological Review*, 34(4), 273–286.
   — Первая формальная теория попарных сравнений в психологии.

2. **Bradley, R. A., Terry, M. E.** (1952). Rank analysis of incomplete block designs: I. The method of paired comparisons. *Biometrika*, 39(3/4), 324–345.
   — Вероятностная модель ранжирования, основа для большинства современных методов.

### Рейтинговые системы

3. **Elo, A. E.** (1978). *The Rating of Chessplayers, Past and Present*. New York: Arco Publishing.
   — Классическая рейтинговая система, адаптируемая для задач оценки качества.

### Межоценщиковое согласие

4. **Cohen, J.** (1960). A coefficient of agreement for nominal scales. *Educational and Psychological Measurement*, 20(1), 37–46. DOI: [10.1177/001316446002000104](https://doi.org/10.1177/001316446002000104)
   — Каппа Коэна для двух оценщиков.

5. **Krippendorff, K.** (2004). *Content Analysis: An Introduction to Its Methodology* (2nd ed.). Thousand Oaks, CA: Sage Publications.
   — Альфа Криппендорфа для произвольного числа оценщиков.

### No-Reference оценка качества изображений (NR-IQA)

6. **Mittal, A., Moorthy, A. K., Bovik, A. C.** (2012). No-Reference Image Quality Assessment in the Spatial Domain. *IEEE Transactions on Image Processing*, 21(12), 4695–4708. DOI: [10.1109/TIP.2012.2214050](https://doi.org/10.1109/TIP.2012.2214050)
   — Метрика BRISQUE. Ключевая работа по безэталонной оценке качества.

7. **Mittal, A., Soundararajan, R., Bovik, A. C.** (2013). Making a "Completely Blind" Image Quality Analyzer. *IEEE Signal Processing Letters*, 20(3), 209–212. DOI: [10.1109/LSP.2012.2227726](https://doi.org/10.1109/LSP.2012.2227726)
   — Метрика NIQE. Не требует обучения — работает «из коробки».

### Нейросетевое ранжирование (Learning to Rank)

8. **Burges, C., Shaked, T., Renshaw, E., Lazier, A., Deeds, M., Hamilton, N., Hullender, G.** (2005). Learning to Rank using Gradient Descent. *Proceedings of the 22nd International Conference on Machine Learning (ICML '05)*.
   — Оригинальная работа по RankNet: обучение нейросети ранжированию через попарные сравнения.

### Архитектуры нейросетей для извлечения признаков

9. **Tan, M., Le, Q. V.** (2019). EfficientNet: Rethinking Model Scaling for Convolutional Neural Networks. *Proceedings of the 36th International Conference on Machine Learning (ICML '19)*. arXiv: [1905.11946](https://arxiv.org/abs/1905.11946)
   — Эффективная архитектура CNN, подходящая как экстрактор признаков для задачи IQA.

### Активное обучение и краудсорсинг

10. **Chen, X., Bennett, P. N., Collins-Thompson, K., Horvitz, E.** (2013). Pairwise Ranking Aggregation in a Crowdsourced Setting. *Proceedings of the 6th ACM International Conference on Web Search and Data Mining (WSDM '13)*.
    — Модель Crowd-BT: адаптивный выбор пар с учётом надёжности каждого оценщика. Прямо применим к твоей задаче.
