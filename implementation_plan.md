**Bradley-Terry реализация (с регуляризацией)**:

> [!WARNING]
> Наивный MLE (без регуляризации) ломается в реальных условиях:
> - Изображение с 0 побед → π = 0 → деление на ноль
> - Изображение без поражений → π → ∞
> - Изолированные кластеры → несопоставимые рейтинги
>
> Решение: **Laplace smoothing** — добавление виртуальных сравнений.

Итеративный алгоритм Зермело-Форда с регуляризацией:
```
Параметр: λ = 0.5 (коэффициент сглаживания)

1. Инициализация: π_i = 1.0 для всех изображений
2. Повторять до сходимости (или max 1000 итераций):
   Для каждого изображения i:
     W_i = число побед i (ничьи = 0.5 победы каждому)
     π_i = (W_i + λ) / Σ_{j≠i} ((n_ij + 2λ) / (π_i + π_j))
   Нормализация: делим все π на среднее
3. Сходимость: max|π_new - π_old| < 1e-6
```

Суть регуляризации: каждая пара изображений получает +2λ «виртуальных» сравнений, а каждое изображение — +λ «виртуальных» побед. Это гарантирует, что:
- Ни один π_i не обнулится (всегда есть хотя бы λ побед)
- Ни один π_i не уйдёт в бесконечность
- Алгоритм сходится даже при 5 реальных сравнениях из 1000 изображений

### 2. API Layer

#### [MODIFY] [api.php](file:///c:/Users/markt/Desktop/Учёба/Диплом/web/api.php)

Новые маршруты:
- `GET ?action=criteria` — список критериев
- `POST action=start_session` — создать сессию (nickname + criterion_id)
- `POST action=vote` — переписан: пишет в SQLite, возвращает comparison ID + номер
- `POST action=undo` — отмена сравнения по ID
- `GET ?action=history&session_id=X` — последние 20 сравнений
- `GET ?action=ratings&criterion_id=X` — рейтинги Bradley-Terry
- `GET ?action=graph&nickname=X&criterion_id=X` — данные графа для оценщика
- `POST action=end_session` — бэкап при уходе
- `GET ?action=pair` — пока случайный, но привязан к сессии
- `GET ?action=image&path=...` — без изменений
