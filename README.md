# Система субъективной оценки качества изображений
## Попарное сравнение (Pairwise Comparison)

### Структура проекта

```
Диплом/
├── generate_test_data.py    # Скрипт генерации тестового датасета
├── README.md                # Этот файл
└── web/
    ├── index.html           # Главная страница SPA
    ├── style.css            # Стили (дизайн-система)
    ├── app.js               # Клиентская логика (Fetch API)
    ├── api.php              # Серверный обработчик (PHP)
    ├── results.log          # Файл лога результатов (создаётся автоматически)
    └── test_data/           # Тестовые изображения (генерируются скриптом)
        ├── noise/           # Цветной шум (20 шт.)
        ├── gradients/       # Градиенты (20 шт.)
        ├── labeled/         # С номерами (20 шт.)
        ├── mixed/
        │   ├── set_a/       # Микс типов (15 шт.)
        │   └── set_b/       # Микс типов (15 шт.)
        └── extra/           # Пиксельный шум (10 шт.)
```

### Быстрый старт

#### 1. Генерация тестовых данных (Python)

```bash
pip install Pillow numpy
python generate_test_data.py
```

Скрипт создаст ~100 изображений в `web/test_data/`.

#### 2. Запуск веб-сервера (PHP)

Из директории `web/`:

```bash
cd web
php -S localhost:8000
```

#### 3. Открытие в браузере

Перейти по адресу: [http://localhost:8000](http://localhost:8000)

### Формат лога результатов

Файл `results.log` содержит записи в формате:

```
[никнейм] [путь_к_изображению_1] [знак] [путь_к_изображению_2]
```

Пример:
```
ivan ./noise/img_003.png < ./gradients/img_012.png
anna ./labeled/img_001.png = ./mixed/set_a/img_007.png
ivan ./extra/img_005.png > ./noise/img_017.png
```

Знаки сравнения:
- `<` — левое изображение **лучше** правого
- `=` — изображения **равны** по качеству
- `>` — правое изображение **лучше** левого

### Горячие клавиши

| Клавиша | Действие           |
|---------|--------------------|
| `1`     | A лучше (левое)    |
| `2`     | Равны              |
| `3`     | B лучше (правое)   |
