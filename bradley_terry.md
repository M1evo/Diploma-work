# Алгоритм Брэдли-Терри в системе попарного сравнения

> Описание используемой модели, реализации и интерпретации результатов.

---

## 1. Постановка задачи

Имеется набор изображений `I = {1, 2, …, N}` и множество субъективных
сравнений вида `(i, j, r)`, где `r ∈ { «<» , «=» , «>» }` — суждение
оценщика о том, какое из двух изображений лучше по определённому
критерию (резкость, контрастность, артефакты, общее качество).

Задача — превратить разрозненные парные суждения в **числовой рейтинг**
каждого изображения, чтобы можно было ранжировать всю коллекцию по
каждому критерию.

---

## 2. Модель Брэдли-Терри

Каждому изображению `i` приписывается параметр «силы» `πᵢ > 0`. Модель
утверждает, что вероятность победы `i` над `j` равна

```
P(i > j) = πᵢ / (πᵢ + πⱼ)
```

Параметры `π = (π₁, …, π_N)` оцениваются методом максимального
правдоподобия. В отличие от Elo, BT пересчитывается **по всем данным
сразу** — порядок поступления сравнений не влияет на результат.

### Учёт ничьих

Ничью `i = j` мы рассматриваем как «полпобеды» каждому, что эквивалентно
расширению Дэвидсона при единичном параметре равенства:

```
W_i ← W_i + 0.5     для каждой ничьи (i, j)
W_j ← W_j + 0.5
```

---

## 3. Итеративный алгоритм Зермело-Форда

Стандартный MM-алгоритм для модели Брэдли-Терри:

```
Инициализация: π_i = 1   для всех i
Повторять:
    для каждого i:
        π_i ← W_i / Σ_{j ≠ i}  n_ij / (π_i + π_j)
    нормализовать π:  π_i ← π_i / mean(π)
Пока  max_i |π_i^(t+1) − π_i^(t)|  >  ε
```

где
- `W_i` — суммарное число побед изображения `i` (с учётом ничьих по 0.5);
- `n_ij` — общее число сравнений между `i` и `j`;
- ε = 10⁻⁶, max_iter = 1000.

---

## 4. Регуляризация (Laplace smoothing)

> [!WARNING]
> Чистый MLE ломается при разреженных данных:
> - изображение без побед → `π = 0` → деление на ноль на следующих итерациях;
> - изображение без поражений → `π → ∞`;
> - изолированные подграфы дают несопоставимые рейтинги.

Решение — **виртуальные сравнения**. Каждой паре `(i, j)` добавляется
`2λ` фиктивных встреч, а каждому изображению — `λ` фиктивных побед:

```
W_i  ←  W_i + λ
n_ij ←  n_ij + 2λ
```

Параметр `λ = 0.5` — компромисс между:
- слишком малым `λ` (~0.01): начальные рейтинги нестабильны;
- слишком большим `λ` (~5): данные почти не влияют, рейтинг плоский.

Эффект:
1. Никакой `πᵢ` не обнулится — гарантирована граница `W_i ≥ λ`;
2. Никакой `πᵢ` не уйдёт в ∞ — знаменатель всегда положителен;
3. Алгоритм сходится даже на разрозненном графе сравнений.

После регуляризации шаг обновления выглядит так:

```
π_i  ←  (W_i + λ) / Σ_{j ≠ i} (n_ij + 2λ) / (π_i + π_j)
```

---

## 5. Нормализация

После каждой итерации π нормируется по среднему:

```
π_i ← π_i / (1/N · Σ_k π_k)
```

Это фиксирует масштаб (среднее = 1), не меняя относительных соотношений.
Удобно для сравнения рейтингов между критериями.

## 6. Реализация в проекте

| Компонент | Файл | Краткое описание |
|-----------|------|------------------|
| Расчёт BT | [web/database.php](web/database.php) → `Database::computeBradleyTerry()` | Чистый PHP, итеративный Zermelo-Ford с λ=0.5 |
| Хранение | таблица `ratings` | поля `bt_score`, `comparison_count`, `updated_at` |
| API | [web/api.php](web/api.php) → `?action=ratings`, `?action=bt_results` | пересчёт по запросу |
| UI | [web/index.html](web/index.html) → экран `#bt-screen` | таблица + сводка |

Константы реализации:

```php
private const BT_SMOOTHING_LAMBDA   = 0.5;
private const BT_MAX_ITERATIONS     = 1000;
private const BT_CONVERGENCE_EPSILON = 1.0e-6;
```

Только **активные** сравнения (`is_undone = 0`) попадают в расчёт —
отменённые оценки полностью игнорируются.

## 7. Интерпретация результатов

В таблице рейтингов каждому изображению присвоено `π`:

- **Среднее ≈ 1.0** — это нормализованный масштаб, а не абсолютный.
- `π = 2.0` — изображение «вдвое сильнее среднего» по критерию.
- Отношение `πᵢ / πⱼ` напрямую даёт коэффициент шансов:
  P(i > j) = πᵢ / (πᵢ + πⱼ).

### Оценка устойчивости

Колонка «Сравнений» в таблице — сколько **активных** сравнений было
проведено по этому изображению (по любому критерию). При малом числе
сравнений (≤ 3) рейтинг слабо отличается от исходных 1.0 — λ-сглаживание
тянет к среднему.

Для надёжной оценки рекомендуется:
- **минимум 10 сравнений на изображение** для каждого критерия;
- **связный граф** — каждое изображение должно быть соединено с
  основной массой через цепочку сравнений (иначе λ-сглаживание тянет
  «изолированные» к среднему искусственно).

## 8. Просмотр текущих результатов

Актуальные рейтинги хранятся в SQLite-базе `web/data/database.sqlite`
(создаётся автоматически при первом запуске). Просмотреть их можно:

1. **В приложении** — кнопка 🏆 в шапке → экран «Рейтинги Брэдли-Терри»
   с выбором критерия. Кнопка «Пересчитать» запускает свежий расчёт по
   всем активным сравнениям.

2. **Через API**:

   ```
   GET /api.php?action=bt_results&criterion_id=1
   ```

   Возвращает JSON со статистикой и списком рейтингов:

   ```json
   {
     "criterion": { "id": 1, "name": "Общее качество", "description": "…" },
     "algorithm": {
       "name": "Bradley-Terry (Zermelo-Ford with Laplace smoothing)",
       "lambda": 0.5,
       "epsilon": 1e-6,
       "max_iterations": 1000,
       "normalization": "mean = 1"
     },
     "stats": { "images": 12, "comparisons": 45, "by_result": {"<": 18, "=": 4, ">": 23} },
     "ratings": [
       { "image_path": "./labeled/img_001.png",
         "bt_score": 2.143, "comparison_count": 9, "updated_at": "…" },
       …
     ]
   }
   ```

3. **Граф сравнений** (кнопка 📊) дополняет таблицу визуальным
   представлением: размер вершины ∝ рейтингу, цвет ребра = результат.
